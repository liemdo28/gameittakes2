# SYSTEM BLUEPRINTS (HIGH LEVEL)

## A) Ability Pairing Loop
1. Player A activates primer ability
2. Environment enters transient state
3. Player B confirms/anchors state
4. System validates co-op condition
5. Reward event: path open / enemy stagger / cinematic trigger

## B) Co-op Sync Score
- Internal metric updated each encounter:
  + timely assists
  + successful chain actions
  + fail-recover efficiency
- Drives adaptive music intensity and optional VO banter variants

## C) Checkpoint Model
- Micro checkpoint at puzzle cluster boundaries
- Encounter checkpoint before major boss phase transitions
- Shared revive tokens per chapter

## D) Failure Recovery
- Detect repeated fail loops
- Offer subtle assist (longer timing, visual hint, VO cue)
- Never auto-solve core puzzle unless accessibility hard mode enabled

## E) Content Streaming
- World Partition cells with chapter-specific data layers
- Preload adjacent encounter bundles
- Audio prefetch for upcoming cinematic triggers
